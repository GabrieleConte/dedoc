Структура распознанного документа

<- на главную страницу 

Возвращается структура ParsedDocument, структура возвращается в формате Json.

ParsedDocument

  1. metadata: DocumentMetadata (обязательное поле) - метаинформация документа
  2. content: DocumentContent (обязательное поле) - разобранная структура документа
  3. attachments: List[ ParsedDocument ] (необязательное поле) - вложенные документы, возвращается только если задано условие обработки вложенных файлов.

DocumentMetadata

Содержит метаинформацию документа (например, Имя файла, размер, времена доступа и т.д.)

  1. file_name: string (обязательное поле) - имя файла (пример: "example.odt")
  2. size: integer (обязательное поле) - размер файла в байтах (пример: 20060)
  3. modified_time: integer (обязательное поле) - дата модификации документа в формате UnixTime (пример: 1590579805)
  4. created_time: integer (обязательное поле) - дата создания документа в формате UnixTime (пример: 1590579805)
  5. access_time: integer (обязательное поле) - дата доступа к файло в формате UnixTime (пример: 1590579805)
  6. file_type: string (необязательное поле) - mime-тип файла (пример: "application/vnd.oasis.opendocument.text")

DocumentContent

Описывает структуры содержимого документа

  1. tables: List[Table] (обязательное поле) - список таблиц документа
  2. structure: TreeNode (обязательное поле) - древовидная структура документа

Table

Обнаруженные и распознанные таблицы в документе

  1. cells: List[List[string]] (обязательное поле) - список списков ячеек таблицы. Ячейка содрежит текст.
  2. metadata: TableMetadata (обязательное поле) - метаинформация таблицы. Может быть пустой.

TableMetadata

Описывает метаинформацию таблицы. Например ее местоположение в документе.

  1. page_id: integer (необязательное поле) - номер страницы с которой начинается таблица.

TreeNode

Описывает узел в древовидной структуре документа.

  1. id : string (обязательное поле) - идентификатор элемента документа. Уникален в рамках одного дерева (т.е. в этом дереве не будет другого такого id, а в attachment могут встретиться повторяющие id) Идентификатор имеет вид 0.2.1 где каждое число символизирует порядковый номер на соотвтетствующем уровне иерархии.
    Например id 0.2.1 означает что этот элемент это вторая подглова третьей главы (нумерация произвдится с 0, первое число это корень документа);
  2. text: string (обязательное поле) - текст элемента;
  3. annotations: List[ Annotation ] (обязательное поле) - поле описывает какие-либо свойства текста, например жирность, размер шрифта и т.д. На данный момент поле пустое.
  4. metadata: ParagraphMetadata (обязательное поле) - метоинформация, относящаяся ко всему субпараграфу, например номер страницы и положение на этой странице.
  5. subparagraphs: List[ TreeNode ] (обязательное поле) - "дети" текущего элемента (например подглавы для главы). Структура "детей" аналогична текущей.

Annotation

Описывает аннотации к тексту, например жирность, курсив, размер шрифта и т.д.

  1. start : integer (обязательное поле) - индекс начала аннотации.
  2. end : integer (обязательное поле) - индекс конца аннотации.
  3. value : string (обязательное поле) - значение аннотации.

ParagraphMetadata

Описывает метаинформацию параграфа (Например это заголовок или елемент списка и т.д.)

  1. type : str (обязательное поле) - тип параграфа (параграф, элемент списка и так далее). Может быть paragraph, list, list_item, raw_text, named_header.
  2. page_id : integer (необязательное поле) - страница на которой начинается этот параграф.
  3. line_id : integer (необязательное поле) - номер строки, на которой начинается этот параграф.
